OpenAI says it built its first AI chip with Broadcom

Cordyceps CI/CD pattern puts 300+ major GitHub repos at risk

Novee Security described Cordyceps, a CI/CD workflow weakness where untrusted pull-request data crosses into privileged workflows, allowing unauthenticated users to run code, steal credentials, or gain repository control. Its scan of about 30,000 high-impact repositories found more than 300 fully exploitable cases, with examples involving Microsoft Azure Sentinel, Google’s adk-samples, Apache Doris, Cloudflare Workers SDK, and Python’s Black. The uncomfortable agent-era angle is that AI-generated workflow changes can reproduce these trust-boundary bugs at scale while each individual YAML step still appears to be “working as designed.”

Figma and Genspark push design tools closer to runnable code

Figma’s latest update adds code layers on the collaborative canvas, repo cloning, code-to-design flow extraction, animation and 3D transform support, AI shader fills, prompt-created plugins, and assistant skills that can use context from tools like Notion, Granola, Excel, and GitHub. Genspark separately launched Genspark Design, powered by Claude Opus 4.7, for generating UI prototypes, videos, HTML animations, and posters, with uploaded Figma files/design systems for brand reuse and one-click code generation through Genspark Code.

Notion introduced External Agents for Claude and Cursor, letting teams @-mention agents, assign work from shared boards, and watch task execution inside the same workspace humans already use. The product pattern is notable because agents become visible collaborators with shared task state rather than private side-channel chats.

Agent tools get real phones and lightweight browsers

MobAI launched tooling that lets Claude Code, Codex, Cursor, OpenCode, Aider, Cline, and other agents drive iOS or Android apps on real devices or simulators via MCP, Python harness, CLI, or HTTP API, with log reading, plain-English test generation, replayable .mob scripts, and demo video capture. Separately, wb is an open-source macOS 26+ browser CLI for agents: an 876KB-to-under-2MB native binary with persistent sessions, compact JSON, screenshots, clicks/forms, JavaScript eval, and live preview, but no Linux, Windows, or older macOS support.

Ray Serve LLM on GKE claims 5x throughput and 8x latency gains

Google Cloud and Anyscale say Ray Serve LLM on Google Kubernetes Engine now reaches up to 5x higher throughput and 8x lower latency through HAProxy-based internal routing, direct token streaming from model replicas, and a v2 Ray executor backend for vLLM. The benchmarks used Gemma 4 E2B on GKE clusters with Google Cloud A4 VMs powered by NVIDIA HGX B200 systems, aiming to close the gap between Ray’s developer ergonomics and native vLLM serving performance.

ParallelKernelBench tests where LLMs fail on multi-GPU kernels

Together AI researchers introduced ParallelKernelBench, a benchmark of 87 multi-GPU kernel problems drawn from real systems including Megatron-LM, DeepSpeed, DeepEP, TensorRT-LLM, and NeMo-RL. The point is narrower and more useful than generic coding scores: models that can write fast single-GPU kernels still struggle with distributed synchronization, communication, and performance constraints.
