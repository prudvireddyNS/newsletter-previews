OpenAI previews GPT-5.6 Sol with restricted cyber-focused rollout

OpenAI previewed GPT-5.6 variants Sol, Terra and Luna to a small set of government-approved companies, with broader availability planned in the coming weeks and a July Cerebras launch mentioned by Andrew Feldman. Gemini 3.5 Flash adds native computer use for browser, mobile, and desktop agents

Google DeepMind says computer use is now a built-in tool in Gemini 3.5 Flash, moving the capability from a standalone Gemini 2.5 computer-use model into the main Flash API and Gemini Enterprise Agent Platform. The model can see, reason, and act across browser, mobile, and desktop interfaces for software testing and knowledge-work automation.

Claude Code and Amazon Q bugs show the repo-trust problem for coding agents

Two disclosures showed AI coding tools turning repository-controlled state into executable behavior. A Claude Code sandbox escape used prompt injection and Git worktree path confusion to overwrite `~/.zshenv`, while Amazon Q Developer patched a high-severity MCP config flaw where a malicious `.amazonq/mcp.json` in a trusted workspace could launch local processes inheriting AWS credentials.

OpenAI says it built its first AI chip with Broadcom

OpenAI announced Jalapeño, its first in-house AI chip, saying it was designed with Broadcom for the LLM workloads behind ChatGPT, Codex, the API, and future agentic products. The company did not disclose specs, deployment timing, or whether the chip targets training, inference, or both.

Ornith-1.0 coding models land in Ollama with MIT-licensed open weights

Ollama posted run commands for Ornith after the Ornith-1.0 family was described as 9B Dense, 31B Dense, 35B MoE, and 397B MoE agentic coding models released under the MIT license. The quoted release claims strong coding results across Terminal-Bench and SWE-Bench variants, with local testing reported for the 35B MoE at FP8 on one DGX Spark.

OpenRouter publishes continuous GPQA and TAU-Bench results for open-weight models

OpenRouter says it continuously runs GPQA and TAU-Bench on most open-weight models and uses those results in AutoExacto, its default meta-benchmark for routing tool calls. Its GLM 5.2 page exposes 1M-token context, reasoning-effort settings, OpenAI-compatible API access, and rolling 30-day provider price, latency, throughput, success-rate, and eval charts.

vLLM shares a four-node DGX Spark recipe for Nemotron 3 Ultra 550B

vLLM highlighted NVIDIA’s guide for serving Nemotron 3 Ultra 550B-A55B-NVFP4 across four DGX Spark machines as one OpenAI-compatible endpoint. The setup uses tensor parallelism size 4, vLLM’s nightly aarch64 OpenAI container, RoCE networking, a 262,144-token max context, Mamba cache flags, and optional MTP speculative decoding.

x.com
↗08

Figma and Genspark push design tools closer to runnable code

Figma’s latest update adds code layers on the collaborative canvas, repo cloning, code-to-design flow extraction, animation and 3D transform support, AI shader fills, prompt-created plugins, and assistant skills that can use context from tools like Notion, Granola, Excel, and GitHub. Genspark separately launched Genspark Design for generating UI prototypes, videos, HTML animations, posters, and code.

techcrunch.com
↗09

Agent runtime tooling converges around harnesses, chat surfaces, and Claude Code orchestration

Vercel’s AI SDK Harness added adapters for LangChain Deep Agents and OpenCode, letting apps swap among Claude Code, Codex, Deep Agents, OpenCode, and Pi through one interface inside Vercel Sandbox. CopilotKit’s OpenTag repo offers a self-hosted alternative to Claude in Slack, while the Claude Protocol adds lifecycle hooks, task worktrees, logged decisions, and PR-based task isolation.

Amazon commits another $13B to AI and cloud infrastructure in India

Amazon says it will invest an additional $13 billion through 2030 to expand AWS data center capacity in Mumbai and Hyderabad, bringing its stated India commitments to $48 billion. The announcement follows major India infrastructure commitments from Microsoft and Google.
