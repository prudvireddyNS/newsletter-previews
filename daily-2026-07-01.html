The workflow uses a main assistant, sub-assistants, user-defined expert assistants, and a fact-checker, and can run on a lab’s own infrastructure while preserving code, environment, plain-language descriptions, and message history for generated figures.

techcrunch.com

Microsoft research warns poisoned MCP tool descriptions can exfiltrate data through agents

Microsoft research described how attackers can hide instructions inside Model Context Protocol tool descriptions so an AI agent leaks data while each individual step still appears authorized. In Microsoft’s invoice example, a third-party enrichment tool secretly tells the agent to collect the last 30 unpaid invoices and attach them to the next tool call, leading Microsoft to recommend treating tool descriptions like system prompts, allowlisting publishers, reviewing description changes, and adding human approval for risky actions.

thehackernews.com

GuardFall bypass hits 10 of 11 open-source AI coding agents tested

Adversa AI’s GuardFall research found shell-command safety checks in 10 of 11 open-source coding and computer-use agents could be bypassed with decades-old bash parsing tricks such as r''m, base64 piping, or destructive flags on ordinary tools. The tested vulnerable tools included opencode, Goose, Cline, Roo-Code, Aider, Plandex, Open Interpreter, OpenHands, SWE-agent, and Hermes, while Continue’s shell-aware parser held in its default editor mode.

thehackernews.com

BioShocking attack tricks AI browsers into leaking signed-in credentials

LayerX’s BioShocking prompt-injection demo tricked six AI browsers and assistants, including ChatGPT Atlas, Perplexity Comet, and Anthropic’s Claude browser extension, into copying credentials from a signed-in GitHub repository and sending them to an attacker. TheHackerNews says OpenAI fixed Atlas while Perplexity closed the report without action and Anthropic’s attempted patch did not hold, while Ars Technica described the root problem as malicious pages lulling agent browsers into an alternate game context where guardrails stop applying.

thehackernews.com

Study finds 282 iOS AI apps leaking LLM API keys or proxy access

Wake Forest researchers tested 444 iPhone AI chatbot apps and found 282 exposed paid AI access in network traffic, including 54 plaintext keys, 92 unauthenticated backend relays, and 136 replayable tokens. Three months after disclosure, only 28% had clearly fixed the leaks, with one popular app issuing a token valid until 2125 and another one-hour token still working 128 days after expiry.

thehackernews.com

Langflow RCE exploited to deploy Monero miner on exposed AI app endpoints

Trend Micro observed attackers exploiting Langflow CVE-2026-33017, a CVSS 9.3 unauthenticated remote-code-execution flaw, against exposed AI application endpoints during a March 27 to April 15 campaign. The payload uses one line of Python to fetch a shell script and Go ELF binary named lambsys, then kills rival miners, disables AppArmor, UFW, iptables, SELinux, and other controls, adds cron persistence, spreads through reused SSH keys, and fetches a custom XMRig miner.
