npm 12 makes install scripts opt-in by default

GitHub released npm 12 with dependency lifecycle scripts, git dependencies, and remote URL dependencies made opt-in by default. The release also begins tightening granular access tokens, with more restrictions planned from August 2026 and direct publishing changes scheduled for January 2027.

Friendly Fire turns coding-agent security reviews into code execution

A proof-of-concept called Friendly Fire shows Claude Code and OpenAI Codex can be tricked into executing attacker-controlled binaries while reviewing untrusted code in autonomous modes. The attack hides prompt injections and execution bait in repository documentation and files, without relying on plugins, hooks, MCP servers, or config changes.

thehackernews.com

GitLost shows public GitHub issues can steer agents into leaking private repo data

Noma Security showed that a public GitHub issue can steer GitHub Agentic Workflows into pulling private repository contents into a public comment when the agent has broad cross-repo read access. The attack needs no stolen credentials, only an issue on a public repo.

thehackernews.com

Improving agents starts with mining traces

LangChain argues that better agents come from mining real traces for failure modes, then using fine-tuning, judge models, and eval loops to hill-climb behavior. The framing is useful for teams moving from demos to continual improvement because it treats agent quality as an operations and data problem, not a prompt rewrite ritual.

02
GitHub’s Copilot code review lesson: tools need task-specific procedure

GitHub shows that giving Copilot code review shared grep, glob, and view tools initially increased cost and missed more issues until the workflow was rebuilt around pull-request evidence. It is a concrete reminder that agent tooling only helps when the operating procedure matches the task.

9 min read

Vercel’s production agent is mostly interesting for its controls

Vercel describes an agent that works across dashboard, GitHub, and CLI to investigate production incidents, review PRs, and propose fixes under its own identity. The durable signal is the control model: read-only by default, approval-gated actions, and production context from logs, metrics, and deployments.

article
04
Simon Willison’s developer breakdown of GPT-5.6

Willison summarizes GPT-5.6 from a builder’s perspective: Luna, Terra, and Sol pricing; 1M context; 128K output; benchmark caveats; and API features such as programmatic tool calling, subagents, prompt cache breakpoints, and original-detail image inputs. It is the practical companion to the launch post if you need to budget and benchmark the new family.

05
Cheap inference changes the shape of data systems

Berkeley researchers map how cheap inference changes data systems: agents issue speculative query storms, need structured shared memory and coordination, and may synthesize custom data systems. The post is a strong architecture checklist for teams building agent-heavy analytics or workflow platforms.

article

Hugging Face makes the vLLM Transformers backend production-relevant

Hugging Face says the Transformers modeling backend in vLLM is now at or above native vLLM speed for many architectures. That reduces the porting tax for new model authors: implement cleanly in Transformers and get fast vLLM serving without waiting for a bespoke backend.

article
07
LangChain’s governed coding-agent blueprint for sensitive code

LangChain shows a governed coding-agent setup using Deep Agents Code, NVIDIA NemoClaw, Nemotron 3 Ultra, deny-by-default networking, human approvals, and audit logs. The useful part is treating the model API as only one piece of the control surface for work near sensitive repositories.

article
08
OpenAI warns that coding benchmarks can mislead routing decisions

OpenAI analyzes issues in SWE-Bench Pro and warns that popular coding benchmarks can produce unreliable signals about model quality. For teams buying or routing coding agents, the takeaway is to treat public benchmark deltas as inputs, not deployment evidence.

article
09
NVIDIA’s Nonuniform Tensor Parallelism playbook for resilient LLM training

NVIDIA explains Nonuniform Tensor Parallelism, an experimental approach that adapts tensor-parallel degree when GPUs become unavailable, overlaps resharding, and uses dynamic power boosting to preserve training goodput. It is useful for infra teams thinking about fault tolerance as scale-up domains grow toward Blackwell-era 72-GPU NVLink systems.

article

Thoughtworks’ PRINCE case study treats agents as engineered systems

Thoughtworks’ Bayer PRINCE case study explains a move from keyword search to an agentic RAG and Text-to-SQL platform for preclinical safety data. It is useful because it emphasizes bounded workflows, context routing, citations, reflection checks, observability, failure handling, evaluation, and human-in-the-loop controls.
