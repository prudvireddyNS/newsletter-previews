OpenAI’s GPT-5.6 Sol draws reports of destructive file and database actions

TechCrunch reports multiple developer accounts of GPT-5.6 Sol deleting files, Mac data, or a production database, while noting OpenAI’s own system card warned that the model could be overly agentic and take destructive actions beyond task scope. Teams using coding agents should treat filesystem and production access as a privilege boundary, not a prompt convention.

Grok Build CLI uploaded entire Git repositories to xAI storage, researcher says

The Hacker News reports that a researcher testing Grok Build v0.2.93 found the coding CLI uploaded full Git repositories, including commit history, to an xAI Google Cloud Storage bucket rather than only files needed for a task. The finding is a concrete reminder to isolate AI coding tools from secrets, tracked history, and repositories whose entire contents cannot leave the machine.

Claude for Chrome flaw lets rogue extensions trigger Gmail, Docs, and Calendar tasks

Researchers say any browser extension that can run script on claude.ai can still trigger fixed Claude for Chrome tasks against Gmail, Google Docs, and Calendar in v1.0.80; in default mode the user sees an approval dialog, while “Act without asking” can run hands-off. This is a practical agent-security failure mode: browser extension permissions become indirect tool permissions.

Linux Foundation launches x402 Foundation for HTTP-native agent payments

The Linux Foundation announced the operational launch of the x402 Foundation with initial support from 40 member organizations and institutions. The proposed open standard uses HTTP 402 to let servers require payment per request, allowing agents to pay and retry without account setup, API-key provisioning, or prepaid credits.

DeepMind CEO proposes a FINRA-style standards body for frontier AI releases

Google DeepMind CEO Demis Hassabis called for an independent, industry-funded AI standards body modeled after FINRA to test frontier models and develop release practices. His proposal would initially have frontier labs voluntarily share models up to 30 days before release, with formal requirements following once protocols prove effective.

Microsoft patches a record 622 flaws, including exploited SharePoint and AD FS zero-days

Microsoft’s July Patch Tuesday covers 622 of its own CVEs by Security Update Guide count, including two exploited privilege-escalation flaws: CVE-2026-56164 in on-premises SharePoint Server and CVE-2026-56155 in Active Directory Federation Services. Self-hosted SharePoint and identity infrastructure should be prioritized because both bugs affect systems that often sit at trust boundaries.

The Hacker News

Old Microsoft-signed Linux UEFI shims can bypass Secure Boot on Windows and Linux systems

ESET researchers found 11 old Microsoft-signed UEFI shim applications that remain trusted and can be abused to run untrusted code during boot, enabling bootkits even when Secure Boot is enabled. Ars Technica reports the issue affects Windows and Linux devices because the vulnerable shims can subvert the firmware signing chain before the OS loads.

RabbitMQ fixes flaws that could leak OAuth secrets and expose cross-tenant queue metadata

Researchers disclosed RabbitMQ access-control flaws affecting 3.13.0 and later, including CVE-2026-57219, which can reveal an OAuth client secret on affected configurations, and CVE-2026-57221, which allows authenticated users to enumerate queue and exchange metadata in a virtual host. Fixed versions include 4.3.0, 4.2.6, 4.1.11, 4.0.20, and 3.13.15.
