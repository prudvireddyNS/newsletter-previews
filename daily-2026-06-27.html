Claude Code and Amazon Q bugs show the repo-trust problem for coding agents

Two new disclosures underline the same failure mode: AI coding tools are turning repository-controlled state into executable behavior. A Claude Code sandbox escape, CVE-2026-55607, used prompt injection and Git worktree path confusion to overwrite `~/.zshenv` and get host code execution even from read-only Bash mode with the full sandbox enabled; Anthropic fixed it in Claude Code 2.1.163. Separately, Amazon Q Developer patched CVE-2026-12957, a high-severity MCP config flaw where a malicious `.amazonq/mcp.json` in a trusted workspace could launch local processes inheriting AWS credentials, with AWS recommending Language Servers for AWS 1.69.0 and updated IDE plugins.

U.S. model-release controls hit OpenAI GPT-5.6 and partially lift for Anthropic Mythos 5

OpenAI says it is limiting the GPT-5.6 lineup—Sol, Terra, and Luna—to a small group of trusted partners whose participation has been shared with the U.S. government, while warning that this access-review process should not become the default for frontier model launches. In parallel, the Commerce Department reportedly allowed Anthropic to redeploy Claude Mythos 5, its cybersecurity-focused model, to more than 100 U.S. companies and agencies, including non-American employees at those organizations; Fable 5 remains unresolved.

Agent runtime tooling converges around harnesses, chat surfaces, and Claude Code orchestration

Vercel’s AI SDK Harness added adapters for LangChain Deep Agents and OpenCode, letting apps swap among Claude Code, Codex, Deep Agents, OpenCode, and Pi through one interface inside Vercel Sandbox. CopilotKit’s OpenTag repo offers a self-hosted, open-source alternative to Claude in Slack, with generative UI, streaming replies, human approvals, full thread context, and adapters for Slack, MS Teams, Discord, Telegram, and WhatsApp. The Claude Protocol adds an enforcement layer for Claude Code using Beads CLI, 13 lifecycle hooks, per-task Git worktrees, logged decisions, and PR-based task isolation.

NVIDIA publishes a GLM-5.2 NVFP4 checkpoint ready for vLLM on Blackwell

NVIDIA’s official `nvidia/GLM-5.2-NVFP4` checkpoint is now servable with `vllm serve nvidia/GLM-5.2-NVFP4`, targeting Blackwell systems with vLLM and SGLang support. The quantized ZAI GLM-5.2 MoE model has 753B total parameters, 40B active parameters, up to a 1M-token context, an MIT license, and claims FP8-level accuracy with a smaller memory footprint across reasoning, coding, long-context, and agentic tool-use evaluations.

Artificial Analysis launches AA-Briefcase for long-horizon knowledge-work agents

Artificial Analysis introduced AA-Briefcase, a private benchmark for agentic knowledge work spanning four multi-week business projects, thousands of input files, and 91 tasks that produce deliverables such as spreadsheets, presentations, and memos. NVIDIA says Nemotron 3 Ultra ranks among the top open models on the leaderboard, which combines rubric pass rate with pairwise Elo scores for analytical quality and presentation.

Cognee v1.0 pitches a long-context memory layer for existing agents

Cognee released v1.0 as an agent memory and retrieval layer rather than a place to build agents, claiming a 100B-token context window, 350ms cold starts, 260ms searches, and lower cost than GPT-5.5 and Claude Opus 4.8 in its own framing. The useful signal is the product direction: agent vendors are increasingly competing on persistent memory, retrieval quality, latency, and token-cost reduction instead of only model choice.
