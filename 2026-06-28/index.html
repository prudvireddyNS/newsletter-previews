Asian AI startups pitch Mythos alternatives while U.S. controls bite TechCrunch reports that Sakana AI launched Fugu in Japan and China’s 360 unveiled Tulongfeng, both positioned against Anthropic’s restricted Mythos/Fable line as U.S. export controls continue. Sakana frames Fugu as an “orchestration model” for coordinating other models and reducing dependency on any single U.S. provider, while 360 is emphasizing automated vulnerability discovery and cyber defense as strategic capabilities. For builders outside the U.S., the practical takeaway is that model-access risk is becoming a product requirement, not just a policy concern.
GLM’s RL post-training stack, slime, is now open source A detailed post describes slime as the RL framework behind GLM-5.2 and earlier GLM releases, with full GLM-5.2 post-training reportedly completed in about two days. The design keeps one training kernel while moving task variety into data generation, wiring Megatron training to SGLang rollout through a Data Buffer and supporting agentic loops, verifier rewards, sandbox interaction, Qwen3, DeepSeek V3, and Llama 3.
Local coding-model demos push long context and small-model distillation One local-agent demo runs Ornith’s 35B MoE coding model at FP8 in vLLM on a single DGX Spark, claiming about 36 tokens/sec with headroom for more than 3 million tokens of context, after a Q4 llama.cpp setup hit about 78 tokens/sec. Another shows Qwythos 9B trained on Claude Mythos traces, served locally on an A6000 with llama.cpp and MTP draft speculation, with claimed 1M-token context via YaRN, native function calling, and 3-of-4 bugs found in a broken full-stack app.
04 Hermes Agent adds Mixture-of-Agents presets Nous Research’s Hermes Agent now exposes MoA presets as virtual models, using multiple reference models plus one aggregator model. The references provide private suggestions without tool access, while the aggregator sees those notes, makes tool calls, and returns the final response, so existing Hermes features such as /goal can keep working.
prd-taskmaster turns Claude Code goals into PRDs and task graphs prd-taskmaster is a pre-alpha MIT-licensed engine for Claude Code that takes a one-line goal, runs a discovery interview, writes a graded PRD, and compiles it into a dependency-ordered task graph. The linked project emphasizes deterministic gates such as placeholder checks, validate-tasks, evidence requirements, and a SHIP_CHECK_OK token, with optional TaskMaster backend support and Python 3.11+ on Linux, macOS, or WSL.
Seedance 2.0 Mini tops Design Arena’s Video Arena Design Arena says BytePlus’s Seedance 2.0 Mini is now No. 1 on its Video Arena with an Elo score of 1342, 13 points above the flagship Seedance 2.0 model. BytePlus models now hold the top three Video Arena slots, ahead of HappyHorse and xAI on that leaderboard.
07 OpenAI ships small Codex UX fixes for long threads OpenAI Devs says Codex received quality-of-life updates this week, starting with smoother scrolling in long conversations. The change keeps the user’s place stable while moving through a thread, a small but useful fix for agent sessions that accumulate many steps and tool outputs.
